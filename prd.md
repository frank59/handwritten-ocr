# 产品需求文档（PRD）：手写表格识别与导出工具

## 1. 产品概述 (Product Overview)
### 1.1 产品名称
手写表格识别与导出工具（Handwritten Table OCR & Excel Exporter）

### 1.2 产品目标
开发一款**纯本地离线**运行的桌面应用，实现从手写单据图片到结构化 Excel 文件的自动转换。核心解决手写文字识别准确率不足的问题，强制引入**人工可视化纠错**环节，确保数据最终准确入库。

### 1.3 目标用户 & 核心场景
*   **用户画像**：需要处理大量手写单据（如采购清单、报价单）的个人或小型企业。
*   **核心痛点**：
    1.  手写字体潦草，自动化工具识别错误率高。
    2.  依赖网络API，数据隐私无法保证。
    3.  笔记本电脑配置有限，需轻量化运行。
*   **核心场景**：用户拍摄手写单据 → 上传电脑目录 → 软件离线识别 → 用户在软件内修正错误 → 导出准确 Excel 对账。

## 2. 核心范围 (Scope)
本产品支持两类输入图片，输出标准 Excel 文件。

### 2.1 支持的输入类型
1.  **类型 A：无框手写清单**
    *   **特征**：无表格线，纯手写排版。通常包含 `品名 + 数量 + 单位`。
    *   *示例*："西红柿 5斤"、"矿泉水 2包"。
2.  **类型 B：印刷表格+手写修改**
    *   **特征**：固定印刷表格模板，部分单元格（如价格、备注）为手写填写或涂改。
    *   *示例*：Excel格式的报价单，品名栏为印刷，单价/金额栏为手写。

### 2.2 最终输出
*   **格式**：标准 `.xlsx` 文件。
*   **结构化字段**：
    *   通用：**原始内容**、**解析金额/数量**、**单位**。
    *   表格类：还原完整的表格行列结构。

## 3. 核心业务流程 (Core Workflow)
`图片输入` $\rightarrow$ `离线OCR识别` $\rightarrow$ `结构化解析` $\rightarrow$ `人工可视化纠错` $\rightarrow$ `Excel导出`

## 4. 详细功能需求 (Detailed Requirements)

### 4.1 图像预处理与输入模块
*   **功能点 4.1.1：图像加载**
    *   支持本地图片文件导入（JPG, PNG, BMP 格式）。
    *   支持批量导入（待确认：是否一期先做单张，二期做批量）。
*   **功能点 4.1.2：自动图像增强**
    *   **自动矫正**：自动检测并矫正倾斜的图片。
    *   **降噪与二值化**：自动去除纸张阴影、褶皱和背景噪点，增强手写文字与背景的对比度。
    *   **分辨率适配**：根据原始图片尺寸进行合理缩放，平衡识别准确率与CPU性能消耗。

### 4.2 离线OCR识别与结构化模块 (核心算法)
*   **功能点 4.2.1：纯离线推理**
    *   所有模型计算均在本地笔记本电脑完成，**无需联网**，不产生任何云端调用费用。
    *   模型需在首次运行时自动下载缓存，后续运行断网可用。
*   **功能点 4.2.2：双场景识别**
    *   **表格识别**：自动检测有线表格的横线/竖线，精准切割单元格，按行列顺序识别内容。
    *   **无框文本识别**：自动分析文字排版区域，区分行块，识别无框手写清单。
*   **功能点 4.2.3：关键字段提取**
    *   **数字与单位抓取**：通过正则表达式识别并提取 `数字`（整数/小数）和 `单位`（斤、包、份、个、元、㎡ 等）。
    *   **数据清洗**：自动剔除识别出的无效符号（如乱码、多余标点），提取纯品名。

### 4.3 可视化人工纠错模块 (关键交互)
*   **功能点 4.3.1：结果预览与编辑**
    *   识别完成后，弹出独立的桌面窗口。
    *   **表格展示**：以表格形式展示识别结果。
    *   **单元格编辑**：支持**双击**任意单元格进行文本修改。
    *   **列结构适配**：
        *   无框清单：自动显示列头 `[品名, 数量, 单位]`。
        *   印刷表格：自动还原列头结构。
*   **功能点 4.3.2：数据增删**
    *   支持手动插入新行（补充漏填项）。
    *   支持删除错误行。
*   **功能点 4.3.3：原图对照** (预留/可选)
    *   界面左侧显示原始单据图片，右侧显示识别结果表格，方便用户对照修正。

### 4.4 Excel 导出模块
*   **功能点 4.4.1：数据持久化**
    *   将人工纠错后的完整数据写入 Excel 文件。
    *   保留数据的原始文本与解析出的数值字段。
*   **功能点 4.4.2：保存对话框**
    *   运行系统原生文件保存对话框，允许用户自定义保存路径和文件名。
    *   支持覆盖已有文件的确认提示。

## 5. 非功能性需求 (Non-Functional Requirements)

### 5.1 运行环境
*   **操作系统**：Windows 10 / Windows 11 (64位)。
*   **硬件配置**：
    *   最低：CPU 双核，内存 8GB。
    *   推荐：CPU 四核，内存 16GB。
    *   *注：强制要求纯CPU推理，不依赖独立显卡。*
*   **运行模式**：**纯离线**。禁止调用任何第三方在线OCR API。
*   **开发环境**：MacOS。

### 5.2 性能要求
*   **识别速度**：单张 A4 尺寸图片的端到端识别时间（含预处理）控制在 **30秒以内**（普通配置下）。
*   **内存占用**：运行过程中峰值内存占用不超过 1.5GB，保证笔记本电脑不卡顿、不死机。

### 5.3 兼容性
*   **支持Docker**：可以通过构建Docker镜像的方式能够在开发机上进行验证测试。
*   **打包部署**：最终交付形式为 **可执行文件 (.exe)**，用户无需安装 Python 环境，双击即可运行。
*   **编码**：全程支持 UTF-8 编码，确保中文手写文字和特殊符号不乱码。

## 6. 技术栈与架构建议 (Tech Stack)
*(此部分供开发人员参考，可作为约束)*
1.  **编程语言**：Python 3.8+。
2.  **核心依赖**：
    *   OCR引擎：PaddleOCR (`PPStructure` 表格结构解析 + `PP-OCRv4` 手写识别)。
    *   图像处理：OpenCV。
    *   GUI框架：PySide6 (轻量稳定，适合打包)。
    *   Excel处理：Pandas + OpenPyXL。
3.  **架构设计**：采用**模块化分层设计**，将 `图像处理层`、`OCR推理层`、`业务逻辑层`、`UI展示层` 解耦，便于后期维护和替换模块。
